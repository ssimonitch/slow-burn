# **Sprint 02: Authentication**

Sprint Duration: 2025-01-27 - 2025-02-02  
Primary Goal: Build complete authentication UI and integrate directly with Supabase using client-side SDK, while ensuring backend API calls include proper JWT authentication.

## **Sprint Overview**

This sprint focuses on implementing the authentication foundation for the Slow Burn AI Fitness Companion. We'll build the login and sign-up pages, implement session management, and create protected routes to ensure secure access to user-specific features.

## **Technical Scope & Boundaries**

### **In Scope:**
- Login and Sign-up UI components with form validation
- Password reset flow UI
- Direct integration with Supabase Auth using supabase-js client
- Session state management using Zustand
- Protected route implementation with React Router
- JWT token management for backend API calls
- Token persistence and auto-refresh logic via Supabase SDK
- Basic error handling and user feedback
- Responsive design for mobile and desktop

### **Out of Scope:**
- Social authentication (OAuth providers)
- Two-factor authentication
- Email verification UI (backend handles this)
- User profile management
- Account deletion functionality
- Advanced session analytics

## **Technical Architecture Decisions**

### **State Management Strategy**
- **Zustand** for auth state (lightweight, TypeScript-friendly)
- **localStorage** for token persistence with encryption
- **React Context** only for theme/UI preferences

### **Security Considerations**
- Supabase handles secure token storage and refresh
- JWTs attached to backend API calls via Authorization header
- XSS prevention through React's built-in protections
- Input sanitization on all forms
- Backend validates all JWTs using JWKS public key verification

### **Authentication Integration Pattern**
```typescript
// Supabase client initialization
import { createClient } from '@supabase/supabase-js'

const supabase = createClient(SUPABASE_URL, SUPABASE_ANON_KEY)

// Auth service wrapping Supabase
class AuthService {
  async login(credentials: LoginCredentials) {
    return await supabase.auth.signInWithPassword(credentials)
  }
  
  async signUp(credentials: SignUpCredentials) {
    return await supabase.auth.signUp(credentials)
  }
  
  async logout() {
    return await supabase.auth.signOut()
  }
}

// Backend API client with JWT using TanStack Query + fetch
import { QueryClient } from '@tanstack/react-query'

const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: 5 * 60 * 1000, // 5 minutes
      retry: (failureCount, error) => {
        if (error.status === 401) return false // Don't retry auth errors
        return failureCount < 3
      },
    },
  },
})

// Centralized fetch wrapper with auth
async function authFetch(url: string, options: RequestInit = {}) {
  const { data: { session } } = await supabase.auth.getSession()
  
  const response = await fetch(`${BACKEND_URL}${url}`, {
    ...options,
    headers: {
      'Content-Type': 'application/json',
      ...(session?.access_token && {
        Authorization: `Bearer ${session.access_token}`
      }),
      ...options.headers,
    },
  })
  
  if (!response.ok) {
    throw new Error(`API Error: ${response.status}`)
  }
  
  return response.json()
}
```
